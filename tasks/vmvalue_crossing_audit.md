# VMValue Crossing Points Audit

Generated by Task #9 for the shape-refactor value-abi migration.

## Categories

| Category | Meaning | Migration target |
|----------|---------|-----------------|
| **convert-to-NanBoxed** | Hot-path call convention / stack ops — eliminate VMValue entirely | Remove `from_vmvalue`/`to_vmvalue`, pass NanBoxed natively |
| **convert-to-ExternalValue** | Wire, display, debug, remote — replace VMValue with ExternalValue | Use `nb_to_external` / `external_to_nb` |
| **must-keep** | Snapshot/serialization (currently needs VMValue for HashMap params) | Keep until Phase 4 (VMValue deletion) |
| **test-only** | Test harness convenience — low priority | Migrate last or keep as-is |

### Hot-path VMValue references that still need migration

These are the remaining VMValue references in hot execution paths:

1. **Executor arithmetic** (22 refs) — arithmetic ops still pattern-match VMValue
2. **Executor trait_object_ops** (20 refs) — trait method dispatch
3. **Executor printing** (29 refs) — print format uses VMValue (PrintSpan.raw_value is Box<VMValue>)
4. **Compiler comptime** (34 refs) — compile-time evaluation
5. **Compiler functions** (16 refs) — function compilation
6. **Context variables** (43 refs) — variable get/set still converts through VMValue
7. **Runtime globals** (37 refs) — global variable access

### Structural VMValue dependencies (data structure level)

These VMValue references are embedded in data structures and cannot be removed by changing function signatures alone:

1. **`SimulationCallData.params: HashMap<String, VMValue>`** — shape-value/src/heap_value.rs
2. **`PrintSpan::Value.raw_value: Box<VMValue>`** — shape-value/src/value.rs
3. **`PrintSpan::Value.format_params: HashMap<String, VMValue>`** — shape-value/src/value.rs
4. **`Upvalue` get/set: `VMValue`** — shape-value/src/value.rs

These structural dependencies must be migrated before Task #34 (Delete VMValue) can proceed.
