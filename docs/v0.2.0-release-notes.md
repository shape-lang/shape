# Shape v0.2.0 release notes

**Released:** 2026-05-17
**Status:** Major architectural rework toward v1.0; pre-1.0 release.
**Predecessor:** v0.1.8 (workspace baseline pre-Phase-3).

v0.2.0 is the first release after the Phase 3 strict-typing migration. The VM and JIT now share a single typed-slot ABI; trait-object dispatch (`dyn T`), bidirectional generic monomorphization, and user-type `Add`/`AddAssign` are working end-to-end across both execution modes. This is an aggressive architectural pre-1.0 — the language semantics are stable, but the user-facing surface (stdlib polish, error messages, tooling, docs) is still maturing toward v1.

## §1 What changed (user-facing)

### §1.1 VM/JIT strict-typing migration

The bytecode VM and Cranelift-backed JIT now share a single typed 8-byte slot ABI driven by per-slot `NativeKind` metadata. Typed opcodes (`AddInt`, `MulNumber`, `EqInt`, etc.) require compile-time proof of operand types; the legacy `ValueWord` tagged-word dynamic-dispatch path is gone (`crates/shape-value/src/value_word.rs` is deleted from runtime code paths; ABI-level tag constants only survive in JIT FFI shim files).

Practical impact for Shape users:
- Arithmetic and field access on annotated types skips runtime type checks.
- `int` and `number` no longer unify implicitly — use `2.0` (not `2`) when a `number` is expected. There is no coercion fallback; a type mismatch is a compile error.
- The `any` escape hatch is gone — every position infers `Type::Variable(TypeVar::fresh())`; if inference fails, it's a compile error.

### §1.2 JIT-side conduit completion across both execution tiers

The §2.7.5 stamp-at-compile-time discipline (kind labels travel through the compiler, not decoded at runtime FFI boundaries) is now operational across the VM↔JIT boundary. Method-dispatch ABI uses `MethodFnV2(&mut VirtualMachine, &[KindedSlot], Option<&mut ExecutionContext>) -> Result<KindedSlot, VMError>`; value-call ABI uses `(callee: KindedSlot, args: &[KindedSlot])`. Frame setup, capture decoding, and the method/value-call paths all carry parallel `NativeKind` metadata so no FFI boundary needs to fabricate a kind from raw bits.

Practical impact:
- The JIT can now optimize closures, methods on user types, generic methods on `dyn T`, and trailing-block continuations without falling back to interpreter.
- Smoke 2 (`[1,2,3,4,5].map(|x|x*2).sum()`) runs at JIT speed end-to-end (was previously an interpreter fallback).

### §1.3 Trait objects: `let t: dyn T = X{}`

`dyn T` dispatch now compiles correctly through the producer/consumer cascade. `HeapValue::TraitObject(TraitObjectPtr)` uses the canonical Path B `#[repr(transparent)]` newtype pattern shared with `HeapValue::TypedObject(TypedObjectPtr)`. VTable thunks emit per-method; the Self-arg vtable-identity check is enforced at `invoke_dyn_unified`.

Practical impact:
- Smoke 5 (`let t: dyn T = X{}; print(t.name())`) runs cleanly under both VM and JIT.
- The grammar uses Shape's annotation syntax `let t: dyn T = X{}` — there is no `box()` keyword.

Caveats (see §5 Known limitations):
- Generic methods on `dyn T` (Q25.C.3 `TypeInfo` threading) dispatch as Direct without the full per-type-arg specialization.
- The `#[static_only]` opt-out marker (Q25.C.4) is not parsed.

### §1.4 `Add`/`AddAssign` for user types

User-defined types can now `impl Add` and `impl AddAssign`, enabling `+` and `+=` on instances. UFCS dispatch falls back through the trait method registry when a bare operator hits a user type.

```shape
type Vec2 { x: number, y: number }
impl Add for Vec2 {
    method add(self, other: Vec2) {
        Vec2 { x: self.x + other.x, y: self.y + other.y }
    }
}

let mut a = Vec2 { x: 1.0, y: 2.0 }
a += Vec2 { x: 0.5, y: 0.5 }
print(a)  // Vec2 { x: 1.5, y: 2.5 }
```

### §1.5 Internal carrier consolidation (Q25.A + Q25.B SUPERSEDED)

`TypedArrayData` enum + `TypedBuffer<T>` + `AlignedTypedBuffer` are deleted from the runtime. `TypedArray<T>` is now the universal array carrier for proven element types; pointer-element arrays use `*const <X>Obj` newtype payloads. `HashMapValueBuf` is similarly deleted; `HashMapData<V>` per-V monomorphization replaces it.

Practical impact: no language-level surface change, but `Array<T>` operations are uniformly typed end-to-end and Drop dispatch is C-compatible.

## §5 Known limitations in v0.2.0

These are explicitly NOT v0.2.0-blocking; they ship as known-incomplete and are the v0.3 / v1 candidate roadmap.

### §5.1 Generic methods on `dyn T` (Q25.C.3)

`VTableEntry::Generic` dispatch treats generic methods as Direct (single thunk per method, no per-type-arg specialization). The `TypeInfo` struct is never constructed at the call site. Practical impact: generic methods on `dyn T` work but don't dispatch with full type-arg precision. Workaround: prefer concrete-type receivers for hot generic-method paths.

### §5.2 `#[static_only]` opt-out (Q25.C.4)

Not parsed at HEAD; ETO-002 error is unreachable from any current Shape program. No workaround needed because nothing depends on it.

### §5.3 IC devirtualization for `dyn T` (Q25.C.6) + LSP cost-class inlay hints (Q25.C.7)

Both UNCOVERED at HEAD. The JIT inline-cache state machine does not extend to `dyn T` call sites; the LSP doesn't surface Direct/Generic/IC cost classes in inlay hints. Roadmap items for v0.3 perf/tooling work.

### §5.4 22 `jit_print` arms uncovered (per-HeapKind formatting fidelity)

13 of 35 `HeapKind` arms are wired into the per-HeapKind kinded `jit_print` dispatch (per ADR-006 §2.7.5.B). The remaining 22 fall through to the baseline path. Affected `HeapKind` families: Numeric/Temporal (Decimal, BigInt, Temporal, Instant), DataTable/Content, Native-foreign (NativeScalar, NativeView), Pure-discriminator (FilterExpr, Reference, SharedCell), Async (Future, TaskGroup), Matrix (Matrix, MatrixSlice), TraitObject/Closure/TypedObject, ModuleFn. Practical impact: `print()` output formatting fidelity for these heap kinds matches the baseline path, not the per-family customizations.

### §5.5 v2-raw class residuals

Two known SIGABRT classes remain in stress-test territory:
- `length_typed_object_empty`
- `w17_comptime_*` family

Territory NOT empirically isolated; needs follow-up audit if pursued. Both are post-cluster-1.5 v2-raw class residuals; see `CLAUDE.md` Known Constraints entry (updated 2026-05-17) for full provenance.

### §5.6 ~48 pre-existing shape-test failures (10 failure classes)

Tracked as `shape-test-residuals-audit`. Categories: generic-fn instantiation returning `Null`, typed-closure inference regressions, array transformation chains, string `.join`, window functions, array slice/sort/some, destructuring rest. Mix of inference-loss + monomorphization + v2-raw-heap. Per-class triage is v0.3+ scope; see `CLAUDE.md` Known Constraints "Pre-existing shape-test failure clusters" entry.

### §5.7 Compiler gap: return-typeless prelude trait method (Phase 4 imprecision 83)

Return-typeless prelude-imported trait methods poison `Vec.map<U>` monomorphization. Architectural compiler gap tracked as `phase-4-followup-return-typeless-trait-method-monomorphization`. v0.3+ scope.

### §5.8 Phase 4 imprecision 84 residuals

`Arc::from_raw` on v2-raw `TypedObjectStorage` wrong-type recovery flake territory. `ReceiverGuard` mirrors landed at `op_get_field_typed:341` + `op_set_field_typed:608`; remaining residual territory is unknown and tracked post-v0.2.0.
