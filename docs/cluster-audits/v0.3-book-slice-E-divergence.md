## §1 — Methodology

For every in-slice page:

1. Read book page end-to-end.
2. For every code-shape claim (struct field, enum variant, function signature, CLI flag, file path), grep the canonical source at `89857260` for the named symbol.
3. For every numeric claim (tier threshold, memory cap, time cap, payload size, cache capacity, default value), grep for the literal constant and compare against the book.
4. Classify each finding as: **MATCH** (book == source), **DRIFT** (book ≠ source, action: update book), **STALE** (claim describes deleted code), **PLANNED-CORRECT** (book labels as planned; source has Rust scaffold but no Shape-level API), **DEFER-TO-V0.4** (gap in language not in book, defer disposition).
