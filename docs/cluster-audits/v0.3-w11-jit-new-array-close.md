## §1. Root cause analysis

The JIT-MIR `Rvalue::Aggregate` consumer at `rvalues.rs:145-167` requires the
destination `Place::Local`'s `ConcreteType` to be `Array<scalar>` (so the
v2 typed-array fast path in `statements.rs:24-65` activates) or
`Struct(_)/Enum(_)/Option(_)/Result(_,_)/Tuple(_)` (so the TypedObject short-
circuit at `statements.rs:61-65` fires). Reaching the fallback at line 145
means the destination slot's `ConcreteType` was stamped `Void` by the
conduit producer at `crates/shape-vm/src/compiler/helpers.rs:564-799`.

Three distinct producer-side gaps were responsible for the W11-AGG surfaces
in the W3.2-A fixture set, each tracking back to a MIR `Rvalue::Aggregate`
emission whose destination slot is not covered by any `*Store` statement
walker arm in the conduit producer:

| Producer-side site | Surface site | Description |
|---|---|---|
| `mir/lowering/expr.rs:2033-2045` (`Expr::Range`) | comprehension iterable `0..N` | `lower_list_comprehension_expr` calls `lower_expr_to_temp(builder, &clause.iterable)` which unconditionally lowers `Expr::Range { start: Some, end: Some }` as `Assign(temp, Aggregate(start, end))`. The temp slot is borrow-tracking only; its `ConcreteType` is never stamped (no `Range` variant in `ConcreteType`, no `RangeStore` MIR statement). |
| `mir/lowering/expr.rs:237-247` (`Expr::Array` with spread) | `[0, ...a, 4]` array literal | `lower_array_expr` emits `Aggregate([Move(scratch_temp_for_0), Move(scratch_temp_for_a), Move(scratch_temp_for_4)])` followed by `ArrayStore`. The conduit producer's `infer_array_elem_from_operands` (`helpers.rs:1269`) only knows how to recover element kind via the `slot_scalar_kind` pre-pass (which classifies `Assign(s, Use(Constant(_)))` scalar temps); the spread source's temp is a `Move(Copy_of_a)` whose ultimate source `a` is `Array<I64>`, not a scalar — the helper returns `None` and the destination slot stays `Void`. |
| `mir/lowering/expr.rs:1581-1598` (`Expr::IndexAccess` with `end_index`) | `nums[1..4]` slice | The slice `IndexAccess { object, index, end_index: Some }` lowers as `Aggregate([nums, start, end])`. The result is a sub-array of `nums`'s element type, but the conduit producer has no `SliceStore` walker arm — destination stays `Void`. NOT addressed in this close. |

The OTHER fail-classes that surfaced in the same fixtures after the W11-AGG
fixes landed were already SURFACE-and-stop sites in their own sub-clusters
(see §3 residuals below).

## §2. Files changed

### `crates/shape-vm/src/mir/lowering/expr.rs::lower_list_comprehension_expr`

W11-jit-new-array (2026-05-17) special-case mirroring `lower_for_expr`'s
existing `Expr::Range { start: Some, end: Some }` pattern. When the
comprehension clause's iterable is a closed range, lower the start/end
sub-expressions to temps without the wrapping `Aggregate`. The bytecode
pipeline's `compile_comprehension_clauses` already runs
`try_begin_range_counter_loop` (`compiler/loops.rs`) emitting a counter-
incremented loop directly without constructing a `RangeData` heap value;
the MIR mirror was missing.

Per ADR-006 §2.7.5 stamp-at-compile-time: the MIR shape no longer produces
an unclassifiable carrier for the comprehension's iterable bookkeeping;
the borrow tracker still observes the sub-expression reads through normal
`lower_expr_to_temp` paths.

### `crates/shape-vm/src/compiler/helpers.rs::infer_top_level_concrete_types_from_mir_with_resolvers`

W11-jit-new-array (2026-05-17) two-part extension:

1. **`copy_source` pre-pass** (new, ~20 LoC): builds a per-slot
   `Vec<Option<u16>>` map from `Assign(dst, Use(Move|Copy|MoveExplicit(
   Place::Local(src))))` statements, so the spread-operand classifier can
   chase the ultimate source of a `Move(temp)` operand back to a user-
   visible slot whose `ConcreteType` was stamped by its own `ArrayStore`.
   The existing slot-move propagation pass at line ~1070 runs AFTER the
   ArrayStore stamping pass, so it cannot help here — the chain walk has
   to be in the first pass directly.

2. **`infer_array_elem_from_operands` signature extension** (3 params →
   5 params): added `concrete_types: &[ConcreteType]` and `copy_source:
   &[Option<u16>]` parameters; in the `Move/Copy/MoveExplicit(Place::
   Local(s))` arm, when the scalar pre-pass produces `None`, walk the
   copy chain via the new `resolve_copy_chain` helper and project the
   element type when the chain terminates at an `Array<T>` slot. Treats
   spread operands as contributing T per the bytecode pipeline's
   `compile_array_with_spread` semantic.

Per ADR-006 §2.7.5 the source slot's `ConcreteType` IS the proof of
element kind; the chain walk is structural (MIR shape only); no runtime
tag-bit decode, no fabricated default. When the chain terminates at a
non-Array slot the helper returns `None` and the caller falls through
to the normal heterogeneous-operand path (slot stays `Void` →
JIT honest surface-and-stop).

`resolve_copy_chain` (new helper, ~25 LoC) is bounded by
`copy_source.len()` against pathological chains and short-circuits at
the first non-`Void` `ConcreteType` entry.

## §4. Residual surfaces (out-of-scope, structured)

### Class A: W11-AGG residual — non-Range / non-spread Aggregate sites (8 snippets)

The remaining W11-AGG surfaces fire on `Rvalue::Aggregate(vec![l, r])`
emissions from `mir/lowering/expr.rs:1720` — the BinaryOp fallback path
for operators not covered by `lower_binary_op` (`Pow`, `BitAnd`, `BitOr`,
`BitXor`, `BitShl`, `BitShr`, `FuzzyEqual`, `FuzzyGreater`, `FuzzyLess`,
`NullCoalesce`, `ErrorContext`, `Pipe`). The empirical fixtures hit
`Pow` (`a ** b`) and `BitAnd`/`BitOr`/`BitXor`/`BitShl`/`BitShr` (`<<`,
`>>`, `&`, `|`, `^`).

Architectural class: **MIR `BinOp` enum coverage gap**. The MIR `BinOp`
enum at `crates/shape-vm/src/mir/types.rs:373-387` covers only
Add/Sub/Mul/Div/Mod/Eq/Ne/Lt/Le/Gt/Ge/And/Or; the rest fall through the
helper's catch-all `None` arm at `mir/lowering/helpers.rs:297-308` and
produce an `Aggregate(vec![l, r])` whose destination temp has no
`ConcreteType` source.

Fix shape (deferred to follow-up sub-cluster `W11-followup-binop-coverage`):
extend `BinOp` enum with Pow / BitAnd / BitOr / BitXor / BitShl / BitShr
variants, update `lower_binary_op`, and add JIT-MIR consumers in
`crates/shape-jit/src/mir_compiler/rvalues.rs` (mirror existing
`Rvalue::BinaryOp` arm). The Pipe / NullCoalesce / ErrorContext arms
involve different semantics (function call / nullable / Result) and need
their own classification.
