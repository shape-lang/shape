## §1 Capabilities landed

### §1.1 1.53 — workspace/symbol eager indexing

**Before** (`server.rs::symbol` pre-W2.7): only `documents.all_uris()`
(open documents). cmd-T workspace queries returned incomplete results.

**After** (`tools/shape-lsp/src/server.rs:1054`-1099):

1. Open documents first (live editor text wins over disk).
2. `ModuleCache::enumerate_workspace_shape_files(project_root)` then walks
   the workspace tree (cap = `MAX_WORKSPACE_FILES = 4096`; excludes
   hidden + `target` / `node_modules` / `dist` / `build`).
3. `visited: HashSet<PathBuf>` de-dups so an open document is not also
   read off disk.

When `project_root` is absent (no `shape.toml` in workspace folder),
behaviour gracefully degrades to the pre-W2.7 open-documents-only path.

### §1.2 1.69 — workspace/didChangeWatchedFiles

**Handler** (`tools/shape-lsp/src/server.rs:1846`-1888): for each
`FileEvent`, if the path is `.shape`:

- `ModuleCache::invalidate(path)` (W2.6 hook at `module_cache.rs:226`)
  drops the cached module-info entry so the next analysis re-reads from
  disk.
- For `CHANGED` and `DELETED`, also drop the per-`Uri`
  `last_good_programs` snapshot so hover / completions don't keep
  serving the stale AST.
- For `CREATED`, the module cache is empty for that path anyway; first
  access will load fresh.

**Protocol**: no `register_capability` registration is emitted server-
side; the LSP client's `workspace.didChangeWatchedFiles` client
capability handles dynamic registration, and tower-lsp-server routes
the notification to our handler regardless. Editors that require
server-side registration can call `register_capability` later (out of
scope for W2.7).

### §1.3 1.71 — workspace/executeCommand

**Capability registration** (`server.rs:670`-679):
`execute_command_provider: Some(ExecuteCommandOptions { commands:
registered_commands(), .. })` — the helper at `server.rs:2052` returns
the canonical list of command IDs emitted by `code_lens.rs`:

- `shape.findReferences`
- `shape.findImplementations`
- `shape.runTests`
- `shape.debugTests`
- `shape.showAnnotation`
- `shape.showTraitMethod`

**Handler** (`server.rs:1894`-1923): acknowledges every declared
command (logs + returns `Ok(None)` so the client routes follow-up
through the corresponding `textDocument/*` method, which is how
rust-analyzer's lens-driven commands work today). Unknown commands
return `Err(method_not_found)` per LSP spec.

**Test (regression guard)**: `registered_commands_cover_every_code_lens_command_w27`
asserts that every command string emitted in `code_lens.rs` is also in
`registered_commands()`. If a new lens command is added without
mirroring, the test fails — i.e. the handler stays in sync with the
producer.

### §1.4 1.72 — workspace/willRenameFiles

**Capability registration** (`server.rs:685`-705) via
`workspace.file_operations.will_rename` with `FileOperationFilter` glob
`**/*.shape` so non-Shape file renames in mixed workspaces don't
round-trip a request we have no edits for.

**Handler** (`server.rs:1935`-2024):

1. For each `FileRename` pair, parse `old_uri` / `new_uri`, derive
   `old_module_path` / `new_module_path` via
   `module_cache::path_to_module_path` (relative to `project_root`).
   Skip pairs outside the workspace or with non-identifier path
   segments — those produce no rewrite edits and the user fixes broken
   imports by hand.
2. For every open document + workspace `.shape` file (W2.6 enumerator,
   de-duplicated against open docs), scan for `from OLD_MODULE_PATH
   use ...` clauses and emit a `TextEdit` replacing the path with
   `NEW_MODULE_PATH`.
3. Return a single `WorkspaceEdit` aggregating per-URI edit vectors.

**Substring-collision guard**: `collect_import_rewrites` reads the path
token as the longest run of identifier characters and `::` separators
after `from `, so `from foo` does NOT match a `from foobar` import.
Test: `collect_import_rewrites_does_not_match_substring_inside_identifier_w27`.

**Indented imports**: handled by `trim_start()` on each line before the
`from ` prefix check; the resulting edit range accounts for the
original indentation. Test:
`collect_import_rewrites_handles_indented_imports_w27`.

### §1.5 New helper: `module_cache::path_to_module_path`

`tools/shape-lsp/src/module_cache.rs:461` derives the canonical Shape
`from`-clause module path (`foo::bar::baz`) for a `.shape` file
relative to the workspace root. Rejects files outside the workspace,
non-`.shape` extensions, and path segments that are not valid Shape
identifiers (leading digit, `-`, etc.) — the latter is conservative
since Shape's `from` syntax cannot reference such modules in the first
place.

**6 cases tested** (`test_path_to_module_path_w27`):

- root-relative: `foo.shape` → `foo`
- nested: `lib/util.shape` → `lib::util`
- deeply nested: `src/sub/mod.shape` → `src::sub::mod`
- non-`.shape`: rejected
- outside workspace: rejected
- invalid identifier (leading digit / `-`): rejected
- underscore identifiers allowed: `_priv/_x.shape` → `_priv::_x`
