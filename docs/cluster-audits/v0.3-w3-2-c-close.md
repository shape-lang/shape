## §3 Per-page fix summary

### installation.mdx
- Quick-test `fn main() -> int { … 0 }` → `fn main() { … }` (unit return); removes noisy `{"Integer": 0}` trailing JSON.
- Added "Expected output" block + inline `// 30` comment for the two snippets.

### first-query.mdx
Systematic rewrite of 7 of 8 snippets:
- Stripped negative-index claim (`values[-1]`) — neither VM nor JIT support it. Replaced with `values[values.len() - 1]`.
- Replaced `samples.mean()/max()/min()` with `.sum() / .len()` on float arrays — `IntrinsicMean` is unwired; `.min/.max` stack-overflow on TypedArray<f64>.
- Trimmed `.map(...)` chain from filter example (kept filter only).
- Renamed `let data` → `let series` (avoids `data` schema-keyword collision); replaced `.mean()/.std()` with manual `sum / len`.
- Replaced anonymous-literal `let record = {...}; record.score = 99` with typed `type Record {...}` + `let mut record` (anon literals can't grow fields).
- Replaced `summarize(Vec<number>)` with `.len()` (not `.length`) + `.sum()` / division — avoided `.min/.max` to dodge stack-overflow surface.
- Wrapped save-and-run script in `fn main() { ... }` for clean unit return; added explicit "Expected output" block.
- Refined "Key Takeaways" — "Types are inferred, but can be made explicit" → "inferred where possible, and required at function boundaries" (accurate to strict-typing).

### basic-concepts.mdx
Systematic rewrite of 5 sections:
- §3 Objects → "Records and Field Access": typed `Point` + `let mut`; one-line explanation of why anonymous literals don't support field assignment.
- §4 closures: bare `let add = |a, b| a + b` (uninferable) replaced with callsite-context lambda (`nums.filter(|x| x % 2 == 0)`); one-line note on inference requirements.
- §6 Option/Result: removed broken `from std::core::intrinsics use { Some, Ok }` import (Some is not a builtin fn); match-based body using global Some/None/Ok/Err names. Changed `Result<T>` → `Result<T, E>` per slice-B §1.C systemic convention. Forward-cite to /fundamentals/errors/.
- §7 Imports and Modules: stripped `use std::core::state` + `state::capture()` (both forms broken). Replaced with `from std::core::math use { sqrt }` (working named-import pattern). Dropped the `use module_path` / `module::name()` example entirely per NO ARCHAEOLOGY.
- §8 Data Tables: stripped bogus `load_events()` call. Kept `type Event` declaration + inline comment pointing at std::core::csv / table_methods for actual loaders.

### repl.mdx
Full rewrite. Original was a stdin-line transcript with `>>` prompts and `help` / `exit` commands — none of which exist at HEAD. The REPL is a ratatui modal TUI. New chapter sourced from `bin/shape-cli/src/repl/{mod.rs,events.rs,rendering.rs}`:
- Cell gutter markers (`[N]` green / `[N]✓` cyan / `▶` output / `!` error / `#` chart)
- Four modes (Normal / Insert / Command / OutputInspect)
- Vim-style key bindings table (i / a / A / I / o / j / k / g / G / x / r / dd / : / q / h / l)
- Insert-mode commit semantics (Esc commits, Shift+Enter executes-and-opens)
- Command-mode real surface: only `:q` / `:quit` / `:clear`
- "A Typical Session" walk-through

No transcript text remains; no `>>` prompts remain.

### hello-world.mdx
Replaced `parse_age(s: string)` (broken string-cast `(s as int?)?` + `if n < 0 or n > 150` Int64 cmp gap) with `validate_age(age: int) -> Result<int, string>` doing two separate `if ... { return Err(...) }` range checks. Added explicit "Expected output" block. Concepts list refined ("Result-based error handling with Ok/Err"; "with where guards").

### web-request.mdx
Aligned API surface to stdlib HEAD per slice-C audit's six update-book dispositions:

- `stringify(value, false)` 2-arg form per json.shape:78.
- `Result<T, E>` two-arg generic per slice-B §1.C.

- Replaced `await join all { ids.map(|id| fetch_user(id)) }` with synchronous for-loop (async forwards to /fundamentals/async-await/).
